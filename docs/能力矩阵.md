# ida-stdio-mcp 能力矩阵

本文档描述 **headless + stdio** 运行形态下，服务对外承诺的能力边界。

## 状态语义

- `full`：当前能力在该域下可稳定使用。
- `degraded`：能力存在，但结果不是最终形态，调用方必须结合 `warnings` 与 `capabilities` 决策。
- `unavailable`：当前环境无后端或无运行条件。

## 能力矩阵

| 能力 | native | managed | 说明 |
| --- | --- | --- | --- |
| `list_functions` | full | full | 两个域都支持符号级函数枚举 |
| `build_callgraph` | full | degraded | managed 目前基于符号与 IL/反汇编文本 |
| `decompile_function` | full / degraded | degraded | native 优先 Hex-Rays；managed 为 symbolic IL fallback |
| `list_strings` | full | degraded | managed 字符串索引来自 IL/反汇编字面量抽取 |
| `set_types` | full | degraded | 托管写回边界受限，必须结合 `type_writeback_support` |
| `patch_bytes` | full | full | 只依赖地址空间与数据库写回能力 |
| `debugger` | available / unavailable | available / unavailable | 取决于当前调试器后端是否真实可用 |

## 参数约定

为了避免 AI 靠猜测拼参数，项目统一以下输入命名：

- 单地址：`addr`
- 多地址：`addrs`
- 普通查询：`query`
- 批量对象：`items`
- 条件过滤：`filter`
- 模式搜索：`pattern`

工具实际可接受的字段以 `tools/list` 返回的 `inputSchema` 为准；另外也可以通过资源：

- `ida://docs/tools`
- `ida://docs/tool/{name}`

读取精确 schema 和最小输入示例。
