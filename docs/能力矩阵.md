# ida-stdio-mcp V2 能力矩阵

本文档描述 V2 在 **IDA 9.3+ headless + stdio MCP** 形态下对外承诺的能力边界。本分支是破坏式协议面：默认工具面以 AI 工作流为中心，不再把 50+ 底层工具全部暴露给模型。

## AI 工作流能力

| 能力 | 默认 slim | full | expert | 门控 | 说明 |
| --- | --- | --- | --- | --- | --- |
| 工作区状态摘要 | 可用 | 可用 | 可用 | public | `get_workspace_state` 返回 runtime、session、working IDB、最近目标与推荐下一步 |
| 打开样本 | 可用 | 可用 | 可用 | public | `open_target` 完整加载样本和调试符号，创建 `.runtime/sessions/<session_id>/working.i64`；默认不等待全库自动分析 |
| 开局 triage | 可用 | 可用 | 可用 | public | `triage_binary` 默认不构建全量字符串索引 |
| 字符串牵引调查 | 可用 | 可用 | 可用 | public | `investigate_string` 返回字符串、xref、所属函数闭环 |
| 函数解释 | 可用 | 可用 | 可用 | public | `explain_function` 聚合画像、伪代码、调用关系与可选 microcode 线索 |
| 输入到检查点追踪 | 可用 | 可用 | 可用 | public | `trace_input_to_check` 做轻量 xref / 函数关系追踪 |
| 报告导出 | 可用 | 可用 | 可用 | public | `export_report` 面向 AI 复盘 |
| 工作区保存 | 可用 | 可用 | 可用 | public | 默认保存 working IDB；显式 path 才导出到用户路径 |

推荐调用顺序：

```text
get_workspace_state -> open_target -> triage_binary -> investigate_string / explain_function -> export_report
```

## 底层 IDA 能力

| 能力 | native | managed/.NET | 默认 slim | full/expert | 说明 |
| --- | --- | --- | --- | --- | --- |
| 函数枚举 | full | full | 通过 triage/explain 间接使用 | 直接可用 | `list_functions` 等底层工具在 full/expert 暴露 |
| 反编译 | Hex-Rays / 汇编降级 | ilspycmd C# / IL 降级 | `decompile_function` 可用 | 直接可用 | 结果带 representation、backend、confidence |
| 调用图 | full | degraded | 通过 triage/explain 间接使用 | 直接可用 | managed 主要基于符号与 IL/反汇编线索 |
| 字符串 | full | degraded | 定点调查可用 | 全量工具可用 | 默认不主动构建大型原生样本字符串索引 |
| 导入表 | full | partial | triage 中摘要 | `list_imports` / `get_import_at` | 9.3+ 优先 import-by-address 富化 module/name/ordinal |
| 类型/结构体 | full | degraded | explain/report 中摘要 | 直接可用 | 托管写回边界需看质量等级 |
| 写回/补丁 | full | partial | 不暴露 | 需 `--unsafe` | 注释、重命名、类型、补丁、IDAPython 均受硬门控 |
| 调试器 | backend dependent | backend dependent | 不暴露 | 需 `--debugger` | 取决于 IDA 调试后端可用性 |
| microcode 只读线索 | Hex-Rays dependent | unavailable | 通过 explain 可选 | 直接可用 | summary/def-use 标记为 experimental |
| microcode mutation | experimental | unavailable | 不暴露 | `--unsafe --tool-surface expert` | 第一版只做严格门控实验能力 |

## Runtime 与会话状态

| 字段 | 说明 |
| --- | --- |
| `source_path` | 原始样本路径 |
| `working_idb_path` | 当前会话工作 IDB，固定在 `.runtime/sessions/<session_id>/working.i64` |
| `dirty` | 当前工作库是否存在未保存写回 |
| `last_active_tool` | 最近成功活动工具 |
| `recent_targets` | 最近分析过的函数、地址、字符串或路径 |
| `recommended_next_tools` | 服务端基于最近活动给 AI 的下一步建议 |

## 上下文隔离

默认共享上下文适合单 agent 串行分析。启用 `--isolated-contexts` 后：

- session 归属按 `context_id` 隔离。
- 会话相关工具和资源必须提供 `context_id`。
- `get_workspace_state` 只返回当前上下文可见 session。
- `close_target`、`save_workspace` 只能操作当前上下文拥有的 session。

缺少 `context_id` 时服务 Fail-Fast，不回退到共享上下文。
