# ida-stdio-mcp 能力矩阵

本文档描述 **headless + stdio** 运行形态下，服务对外承诺的能力边界。

## 运行模式

服务支持两种上下文模式：

- **共享默认上下文**：默认模式。适合单 agent、串行分析、人工控制切换会话。
- **隔离上下文模式**：通过 `--isolated-contexts` 或 `setting.toml` 中
  `feature_gates.isolated_contexts = true` 开启。适合同一 MCP 进程内的多 agent /
  多工作流并发分析。

### 隔离上下文模式的强约束

开启后：

- 所有会话相关工具都必须显式提供 `context_id`
- 会话 ownership 按 `context_id` 隔离
- `list_binaries` 只返回当前 `context_id` 自己的会话
- `switch_binary` / `close_binary` / `save_binary` / `deactivate_binary`
  只能操作当前 `context_id` 自己的会话
- `ida://sessions` / `ida://session/current` 以及所有 session-scoped resources
  都要求显式提供 `context_id`

如果在隔离模式下缺少 `context_id`，服务会 **Fail-Fast** 返回错误，不会回退到共享默认上下文。

## 状态语义

- `full`：当前能力在该域下可稳定使用。
- `degraded`：能力存在，但结果不是最终形态，调用方必须结合 `warnings` 与 `capabilities` 决策。
- `unavailable`：当前环境无后端或无运行条件。

## 能力矩阵

| 能力 | native | managed | 说明 |
| --- | --- | --- | --- |
| `list_functions` | full | full | 两个域都支持符号级函数枚举 |
| `build_callgraph` | full | degraded | managed 目前基于符号与 IL/反汇编文本 |
| `decompile_function` | full / degraded | degraded | native 优先 Hex-Rays；managed 为 symbolic IL fallback |
| `list_strings` | full | degraded | managed 字符串索引来自 IL/反汇编字面量抽取 |
| `set_types` | full | degraded | 托管写回边界受限，必须结合 `type_writeback_support` |
| `patch_bytes` | full | full | 只依赖地址空间与数据库写回能力 |
| `debugger` | available / unavailable | available / unavailable | 取决于当前调试器后端是否真实可用 |

## 上下文与会话隔离矩阵

| 能力 | 共享默认上下文 | 隔离上下文模式 | 说明 |
| --- | --- | --- | --- |
| `open_binary` | shared | isolated | 隔离模式下，同一路径可被不同 `context_id` 分别打开成不同 session |
| `list_binaries` | global list | context local | 隔离模式下只返回当前 `context_id` 可见的会话 |
| `current_binary` | shared current | context local | 隔离模式下不同 `context_id` 各自维护默认当前会话 |
| `switch_binary` | shared current switch | isolated ownership | 隔离模式下不能切到别的 `context_id` 的 session |
| `close_binary` | global session close | isolated ownership | 隔离模式下只能关闭本上下文拥有的 session |
| `save_binary` | global session save | isolated ownership | 隔离模式下只能保存本上下文拥有的 session |
| `deactivate_binary` | shared default binding | context local binding | 隔离模式下只解除当前 `context_id` 的默认绑定 |
| `ida://sessions` | 全局可见 | 当前上下文可见 | 隔离模式下必须显式提供 `context_id` |
| `ida://session/current` | 全局默认当前会话 | 当前上下文默认会话 | 隔离模式下必须显式提供 `context_id` |
| session-scoped resources | 依赖共享当前会话 | 依赖 `context_id` + 当前上下文会话 | 防止多 agent 并发串台 |

## 参数约定

为了避免 AI 靠猜测拼参数，项目统一以下输入命名：

- 单地址：`addr`
- 多地址：`addrs`
- 普通查询：`query`
- 批量对象：`items`
- 条件过滤：`filter`
- 模式搜索：`pattern`
- 上下文隔离：`context_id`

工具实际可接受的字段以 `tools/list` 返回的 `inputSchema` 为准；另外也可以通过资源：

- `ida://docs/tools`
- `ida://docs/tool/{name}`

读取精确 schema 和最小输入示例。

### `context_id` 使用建议

- 单 agent / 串行任务：可以不开 `--isolated-contexts`
- 多 agent / 并发任务：必须开启 `--isolated-contexts`
- 建议每个 agent / workflow 使用稳定的 `context_id`
  - 例如：`agent-1`、`agent-2`、`triage-worker`
